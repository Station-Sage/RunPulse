# UX 원칙 상세

## 원칙 1: 데이터 통합과 소유권

UI 구현 시:
- 활동 상세 페이지에서 소스 라벨을 기본적으로 숨김
- "소스 보기" 토글 시 각 메트릭 옆에 소스 아이콘 표시
- 소스 간 값이 다를 때: 대표값 표시 + "다른 소스와 비교" 링크
- 데이터 내보내기: SQLite DB 자체를 다운로드 가능하게

## 원칙 2: 투명한 분석

UI 구현 시:
- 모든 RunPulse 계산 메트릭에 (i) 아이콘
- (i) 탭하면: 한 줄 설명 + 계산식 + 입력 데이터 링크
- AI 코치 응답에 "근거 보기" 버튼
- "근거 보기" 탭하면: 참조된 메트릭 목록 + 각각의 현재값

## 원칙 3: 맥락 있는 안내

UI 구현 시:
- 대시보드 상단에 "오늘의 요약" 카드 (AI 생성, 1-2문장)
- 요약 카드 탭하면 AI 코치 대화로 진입
- 훈련 프로그램 활성화 시: "오늘의 세션" 카드가 최우선
- 메트릭이 위험 구간 진입 시: 프로액티브 알림 (왜 알리는지 설명 포함)

## 화면별 첫 3초 설계

### 대시보드 (홈)
첫 3초: "오늘 내 상태" — CRS 기반 한 줄 + 오늘 할 훈련

### 활동 상세
첫 3초: "이 달리기는 어땠나" — 핵심 3개 (거리/시간/페이스) + 평가 한 줄

### 주간 리뷰
첫 3초: "이번 주는 어땠나" — 주간 볼륨 + CTL 변화 + 한 줄 평가

### AI 코치
첫 3초: 코치의 인사 + 오늘 핵심 메시지 1개
