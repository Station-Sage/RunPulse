# v0.2 UI/메트릭 반영 수준 점검 및 보완 사항 정리

## 1. 문서 목적

이 문서는 현재 프로젝트의 구현 상태를 기준으로,
최종 프로토타입 / UI spec / metrics 설계 문서와 비교했을 때 부족한 점과 보완해야 할 내용을 상세히 정리한 것이다.

목적은 다음과 같다.

1. 현재 구현 수준을 todo 완료 기준으로 재판단
2. `./ai` 폴더의 메트릭 정의와 실제 UI 반영 상태를 비교
3. 미구현 / 부분구현 / 해석 부족 / UX 부족 항목을 화면별로 정리
4. Claude 등에 넘길 수 있도록 작업 명세형 TODO 문서로 활용
5. v0.2 검증 완료 후 바로 다음 작업으로 연결되게 함

## 3. 종합 평가

### 3.1 전체 구현 수준
현재 구현은 메트릭 엔진 기준으로는 높은 수준,
UI 반영 기준으로는 중간 수준,
최종 프로토타입 재현 기준으로는 아직 부분 구현 수준이다.

대략적으로 보면:

- 메트릭 계산 엔진 구현률: 높음
- 핵심 화면 기능 구현률: 중상
- 최종 프로토타입 UI/UX 재현율: 중하
- 메트릭 설명/해석 UX 완성도: 낮음~중간
- 보조 탭(레이스/AI/훈련계획/웰니스) 완성도: 낮음

### 3.2 한 줄 결론
현재 상태는
"핵심 데이터와 핵심 화면은 동작하지만, 최종 프로토타입 수준의 제품 경험을 만들기 위한 연결 UI/해석 UX/보조 화면이 크게 부족한 상태"
로 판단된다.

## 4. 최종 프로토타입 대비 구현 수준 분석

### 4.1 구현 강한 영역

#### 4.1.1 Dashboard
강점:
- UTRS, CIRS, RMR, PMC가 메인 구조로 반영되어 있음
- 핵심 시각화가 이미 존재
- 최근 활동 리스트 연결도 존재

판단:
- 기능 수준은 강함
- 다만 프로토타입 수준의 coaching UX는 부족

#### 4.1.2 Activities
강점:
- 활동 리스트, 필터, 그룹핑, 머지 등 실사용 기능이 강함
- source 병합 관리 기능이 프로토타입보다 오히려 운영 측면에서 강점이 있음

판단:
- 데이터 운영 UI는 강함
- 하지만 최종 프로토타입 수준의 시각적 정리와 insight UX는 부족

#### 4.1.3 Activity Detail
강점:
- 활동 단위 deep analysis 구조가 잘 잡혀 있음
- FEARP, GAP, Relative Effort, Decoupling 등 노출
- source별 메트릭 비교가 강함

부족:
- map 미완성
- prototype 지향의 훈련 해석 화면으로는 아직 부족
- DI/DARP/TIDS/LSI 등 핵심 2차 메트릭 노출이 빈약

판단:
- 분석 데이터는 강함
- 프로토타입형 인사이트 카드 UX는 부족

#### 4.1.4 Report
강점:
- 기간 선택, summary, 차트, 메트릭 테이블 존재
- 기본 리포트 틀은 있음

부족:
- TIDS 분포
- TRIMP 중심 부하 분석
- 위험지표 요약
- AI insight
- DARP 예측 요약
- VDOT/Marathon Shape/eVO2max 추세

판단:
- 기본 리포트는 있음
- 훈련 의사결정 리포트로는 아직 약함

### 4.2 구현 약한 영역

#### 4.2.1 Race Prediction
- 사실상 미구현 수준
- DARP 계산은 있으나 UI 연결이 약함 또는 없음
- 거리 선택, 예측 기록, 페이스 전략, split, wall risk 등 부재

#### 4.2.2 AI Coach
- 최종 프로토타입의 핵심 차별화 포인트인데 UI 거의 없음
- daily briefing
- 추천 chip
- 질문 기반 UX
- regenerate
- 대화형 coaching 모두 부족

#### 4.2.3 Training Plan
- 주간 계획 / 달력 / workout 배치 / sync 상태 / AI 추천 연결 부재

#### 4.2.4 Wellness / Recovery
- UTRS 재료 데이터 일부는 엔진에 있으나 별도 wellness 화면으로 정리되지 않음
- sleep / HRV / body battery / stress / readiness 흐름 미시각화

#### 4.2.5 Settings Hub 완성도
- 연결 페이지는 있으나
- prototype 수준의 통합 허브 UX는 아님
- import/export, sync 상태 추적, 사용자 설정, 메트릭 정책 선택, fallback 상태 노출 부족

## 5. 메트릭별 UI 반영 현황

### 5.1 반영이 잘 된 메트릭

#### UTRS
현재 반영:
- Dashboard 핵심 카드
- Activity 일부 daily score
- Report 평균값 일부

평가:
- 대표 메트릭으로 잘 노출됨
- 다만 세부 구성요소 설명은 부족

보완:

- 오늘 추천 훈련과 연결
- 최근 7일 추세 추가

#### CIRS
현재 반영:
- Dashboard 핵심 카드
- Activity 일부 daily score
- Report 평균값 일부

평가:
- 핵심 리스크 메트릭으로 잘 노출됨
- 그러나 risk factor 설명력 부족

보완:
- ACWR / Monotony / Spike / Asymmetry factor breakdown
- 위험 상태 문구 정교화
- action recommendation 추가

#### RMR
현재 반영:
- Dashboard radar

평가:
- 시각화는 좋음
- 하지만 각 축 해석 부족

보완:
- 축별 정의
- 지난 4주/12주 비교
- 낮은 축 개선 제안

#### FEARP
현재 반영:
- Dashboard recent activities badge
- Activity detail
- Report table 일부

평가:
- 노출은 꽤 잘 되어 있음
- 그러나 왜 보정되었는지 설명이 없음

보완:
- temperature / humidity / altitude / grade 영향 분해 UI
- 실제 pace 대비 차이 시각화

#### GAP
현재 반영:
- Activity detail
- Report 일부

평가:
- 기본 노출은 있음
- 개념적 설명 부족

보완:
- actual pace vs GAP 비교
- 업힐/다운힐 영향 표시

#### Relative Effort
현재 반영:
- Dashboard
- Activity
- Report

평가:
- 반영 양호
- zone 기반 설명 보강 필요

#### Decoupling
현재 반영:
- Activity
- Report

평가:
- 메트릭 자체 노출은 됨
- 훈련 해석 UX가 부족

보완:
- good / caution / poor 뱃지
- 전반부/후반부 EF 비교 시각화
- long run suitability 해석 추가

### 5.2 부분 반영된 메트릭

보완:

#### DARP

## 6. 화면별 상세 보완 사항

### 6.1 Dashboard 보완 사항

현재 Dashboard는 핵심 지표는 보여주지만,
"그래서 오늘 어떻게 훈련해야 하는가?" 로 연결되는 제품 경험이 약하다.

반드시 보완할 항목:

1. 오늘의 훈련 권장 카드 추가
   - 오늘 상태 요약
   - 추천 강도
   - 추천 시간/거리 범위
   - 회복 권장 여부
   - 주의 문구

2. UTRS breakdown UI 추가
   - sleep
   - hrv
   - tsb
   - rhr 또는 body battery
   - sleep consistency
   - 각 항목 기여도 bar

3. CIRS breakdown UI 추가
   - ACWR
   - Monotony
   - Spike
   - Asymmetry
   - 각 항목 상태
   - 위험 원인 설명

4. Risk summary pills 추가
   - ACWR 상태
   - LSI 상태
   - Monotony 상태
   - TSB 상태

5. DARP mini card 추가
   - 5K / 10K / Half / Full 예측
   - 최근 추세 상승/하락
   - 기준 근거(VDOT + DI)

6. Fitness mini card 추가
   - VDOT
   - eVO2max
   - Marathon Shape
   - 최근 4주 변화량

7. graceful fallback 강화
   - 최근 90분 이상 러닝 부족 안내
   - 수면/HRV 데이터 부족 안내
   - 심박 존 데이터 부족 안내

### 6.2 Activity Detail 보완 사항

현재 Activity Detail은 데이터는 많지만 훈련 해석 흐름이 부족하다.

반드시 보완할 항목:

3. Decoupling 해석 카드 강화
   - EF first half
   - EF second half

4. Secondary metric card 확장
   - DI

   - Monotony
   - ACWR
   - ADTI
   - TIDS
   - Marathon Shape badge
   - VDOT/eVO2max reference

5. Activity intent / workout classification 표시
   - easy
   - tempo
   - interval
   - long run
   - recovery
   - race

6. map 섹션 구현 또는 placeholder 정리
